### 🛠️ 給 AI 的教訓與準則
- **禁止隨意刪除**：除非是該對話階段中 AI 自己建立的臨時檔案，否則嚴禁對任何既有資料夾執行 `rm -rf`。
- **內容審核**：在執行清理指令前，必須先 `ls -R` 徹底確認目錄內容，並向使用者逐一確認。
- **原始資料至上**：永遠假設使用者提供的原始檔案（如 .epub, .pdf, .raw）是不可再生的。

### 💡 給人類的提醒
- **不要完全信任 AI 的終端機權限**：AI 可能會誤解「清理」的範疇。
- **外部備份**：重要原始檔案應存放在專案目錄以外的安全路徑（如獨立的雲端或備份碟），不要放在開發中的 Git 倉庫內。
- **監督高風險指令**：當看見 AI 準備執行 `rm`, `mv` 或 `git clean` 時，請務必停看聽。
